# Phase 6 тАФ UI Design Contract

> Visual and interaction contract for Hospitality Rendering. Generated by gsd-ui-researcher (2026-04-25). Verified by gsd-ui-checker.
>
> **Scope discipline.** Phase 6 introduces **two net-new components** (`HospitalityCard`, `HospitalitySection`) and **branches an existing screen** (`PropertyDetailsScreen`) тАФ every visual decision below is expressed as a token reference to `useTheme().colors` (loaded from `src/theme/colors.ts`). No new design-system tokens. No new spacing scale. No new typography scale. No new icon library (lucide-react-native already in stack). Decisions locked upstream by 06-CONTEXT.md (D-01..D-24) and 06-RESEARCH.md are NOT re-litigated; they are encoded here as visual contract.
>
> **Token source-of-truth.** `src/theme/colors.ts` exposes `colors.light` + `colors.dark`; both contain identical key sets. The 14 keys in scope for this phase: `background`, `surface`, `text`, `textSecondary`, `textTertiary`, `accent`, `border`, `inputBackground`, `chipBackground`, `chipBorder`, `activeChipBackground`, `activeChipText`, `success`, `error`. Dark/light parity is guaranteed by the file shape itself.

**Preserve existing тАФ do NOT introduce a new type scale.** Three sizes for card body, two weights for chips, plus the existing PropertyDetailsScreen typography for branch points.

| Role | Size | Weight | Line Height | Font | Usage in this phase |
|------|------|--------|-------------|------|---------------------|

| Card body / address | 14px | 400 | platform default | Platform sans | `HospitalityCard.address`, rooms + maxGuests meta line |

| Form amenity chip text | 14px | 500 | platform default | Platform sans | Picker chips in `HospitalitySection` form тАФ matches `commonStyles.chipText` exactly |

| Contact bar button (icon-only) | N/A | N/A | N/A | N/A | Icon-only; lucide icon size=24 for visual weight in a 56x56 button |

**Weight vocabulary:** 400 (regular тАФ body, address, detail amenity), 500 (medium тАФ meta caption, amenity chip), 600 (semibold тАФ type badge, count chip), 700 (bold тАФ strip section title), 800 (extrabold тАФ tour3D badge, preserved from PropertyCard).

**RU-length overflow (MANDATORY for new chips and section headers):**

- `typeBadge` `<Text>`: `numberOfLines={1}` тАФ RU "╨е╨╛╤Б╤В╨╡╨╗" 6 chars vs "Hostel" 6 chars; "╨Ю╤В╨╡╨╗╤М" 5 chars vs "Hotel" 5 chars. **No overflow risk** but apply `numberOfLines={1}` defensively.
- `amenity chip <Text>` (form picker): `numberOfLines={1}` MANDATORY. Russian labels are longer ("╨Ъ╨╛╨╜╨┤╨╕╤Ж╨╕╨╛╨╜╨╡╤А" 11 chars vs "Air conditioning" 16 chars; "╨Ъ╨░╨╝╨╡╤А╤Л ╤Е╤А╨░╨╜╨╡╨╜╨╕╤П" 15 chars vs "Lockers" 7 chars). Without `numberOfLines={1}`, "╨Ъ╨░╨╝╨╡╤А╤Л ╤Е╤А╨░╨╜╨╡╨╜╨╕╤П" wraps the chip onto two lines, breaking the grid rhythm.
- `amenity preview chip <Text>` (card): `numberOfLines={1}` + `ellipsizeMode="tail"` MANDATORY. Card hero is space-constrained (280pt total minus image padding); long RU labels truncate gracefully (e.g. "╨Ъ╨╛╨╜╨┤╨╕╤ЖтАж").
- Strip `title` (`home.hospitalitySectionTitle`): `numberOfLines={1}` defensive. EN "Hostels & Hotels" 16 chars; RU "╨е╨╛╤Б╤В╨╡╨╗╤Л ╨╕ ╨╛╤В╨╡╨╗╨╕" 15 chars тАФ comparable, no overflow risk on 375pt+ screens.
- Strip `count chip text`: never wraps (parens + integer only).
- Detail amenity grid label: `numberOfLines={1}` + `ellipsizeMode="tail"` for long RU labels in 2-column layout.

**Heading/body line-height:** Do not set `lineHeight` explicitly. Inherit platform defaults (matches every other card surface in the app).
