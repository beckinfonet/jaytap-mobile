<objective>
Add the `errors.permissionDenied` i18n key to both `src/locales/en.ts` and `src/locales/ru.ts` per D-18. This is the user-facing translation UI catch sites will use when `PropertyService.patchPlatformVerifications` throws `PermissionDeniedError` (`error.message === 'E_PERMISSION_DENIED'`).

Purpose: Services throw error codes, not translated strings (D-18 translation boundary — services don't have access to `t()`). The UI catches and translates. This plan lands the translation key ahead of the service guard (Plan 04) so downstream UI code (if any UI ever catches this error and displays it) has a valid key to look up.

Output: Two locale files modified with new `errors.permissionDenied` key in both EN and RU. TypeScript's `Record<TranslationKeys, string>` exhaustiveness check enforces parity automatically — if EN has the key and RU doesnt, compilation fails.

This plan runs in Wave 1 in parallel with Plan 02 (no shared files — Plan 02 touches `src/hooks/`, `src/components/`, `src/constants/`; Plan 03 touches only `src/locales/`).
</objective>
