independently if a per-screen verification failure surfaces in Plan 06). One shared import

LOAD-BEARING DETAILS (RESEARCH §3.2):

1. AuthModalCloseButton MUST stay a SIBLING of KASV (NOT inside it) — its absolute-positioned
   tap target would be eaten by KASV's gesture handler otherwise. RESEARCH §3.2 step 3.
2. contentContainerStyle moves the existing centering (justifyContent:'center' from
   styles.container) onto KASV's content container with flexGrow:1 so the layout looks
   identical when the keyboard is closed.
3. keyboardShouldPersistTaps="handled" — needed so tapping the submit button while keyboard
   is open does not require two taps.
4. bottomOffset={20} — library's idiomatic 20pt gap below the focused field. NOT a
   keyboardVerticalOffset magic number; will not trigger §6 SC3.a grep gate. (PATTERNS
   Pattern 1 explicitly notes this distinction.)
5. PasswordTextInput composition (used in LoginScreen, SignupScreen, ResetPasswordScreen)
   requires ZERO changes per CONTEXT D-Discretion + RESEARCH §3.2 step 4. KASV finds the
   focused input via React Native's currentlyFocused() API regardless of nesting.
