<!-- Banner component (RESEARCH.md "Code Examples — RoleRefreshBanner" lines 1064-1106 + PATTERNS.md correction at lines 887-911 about correct import paths and t() usage + W2 stale-closure dismiss fix) -->

PATTERNS.md flag 3 corrects the import paths: `useTheme` is `'../theme/ThemeContext'` (NOT `'../theme/useTheme'`); `useLanguage` is `'../context/LanguageContext'`; `t()` is from `'../locales'` and called as `t(language, key)` — NOT `useLanguage().t(key)`.

**W2 stale-closure dismiss bug correction:** the previous design used `onPress={async () => { await refreshRole(); setLastSeenRole(currentRole); }}`. That captures `currentRole` from the pre-refresh closure — when the role actually changed, `currentRole` (from the closure) would equal `lastSeenRole`, so the banner re-evaluates to "no change pending" only on the NEXT render. Worse, if `refreshRole()` mutates `user.backendProfile.userType` synchronously into the next render, the closure's `currentRole` is stale and `setLastSeenRole(currentRole)` writes the OLD role into lastSeenRole — banner stays visible.

Fix: optimistic dismiss + useEffect rebase. The tap handler immediately sets `lastSeenRole = undefined` (banner hides because the show condition requires both `lastSeenRole` AND `currentRole` truthy). Then `refreshRole()` runs. A `useEffect` watching `currentRole` rebases `lastSeenRole` to the latest value when `lastSeenRole === undefined`. Result: banner hides instantly on tap, refreshRole completes, lastSeenRole catches up.

Final shape:
```typescript
// src/components/RoleRefreshBanner.tsx
import React, { useState, useEffect } from 'react';
import { View, Text, TouchableOpacity, StyleSheet } from 'react-native';
import { useAuth } from '../context/AuthContext';
import { useTheme } from '../theme/ThemeContext';
import { useLanguage } from '../context/LanguageContext';
import { t } from '../locales';

export const RoleRefreshBanner: React.FC = () => {
  const { user, refreshRole } = useAuth();
  const { colors } = useTheme();
  const { language } = useLanguage();
  const currentRole = user?.backendProfile?.userType;
  const [lastSeenRole, setLastSeenRole] = useState<string | undefined>(currentRole);

  // Rebase: when lastSeenRole is undefined (post-tap optimistic dismiss OR initial mount
  // before currentRole is hydrated) and currentRole becomes available, catch up.
  useEffect(() => {
    if (lastSeenRole === undefined && currentRole) {
      setLastSeenRole(currentRole);
    }
  }, [currentRole, lastSeenRole]);

  const showBanner = !!lastSeenRole && !!currentRole && lastSeenRole !== currentRole;
  if (!showBanner) return null;

  return (
    <TouchableOpacity
      style={[styles.banner, { backgroundColor: colors.warning }]}
      onPress={async () => {
        // Optimistic dismiss — banner hides immediately because showBanner
        // requires lastSeenRole truthy. The useEffect above will rebase
        // lastSeenRole to currentRole once refreshRole settles.
        setLastSeenRole(undefined);
        await refreshRole();
      }}
      activeOpacity={0.8}
    >
      <Text style={[styles.text, { color: colors.onWarning }]}>
        {t(language, 'auth.roleChanged.banner')}
      </Text>
    </TouchableOpacity>
  );
};

const styles = StyleSheet.create({
  banner: { paddingVertical: 12, paddingHorizontal: 16 },
  text: { fontSize: 14, fontWeight: '600', textAlign: 'center' },
});
```
