- Single positional arg `address: string`
- Returns `Promise<{ latitude: number, longitude: number } | null>`
- Hard-codes `Accept-Language: en`
- Hard-codes `addressdetails: 0`
- No AbortController, no timeout, no countrycodes, no viewbox

Post-Phase-11 signature MUST be `geocodeAddress(address: string, opts?: { bias?: { lat: number, lng: number }, lang?: string }): Promise<{ latitude, longitude, displayName, addressDetails } | null>` — new return-shape fields `displayName` (string from Nominatim's `display_name`) and `addressDetails` (object from Nominatim's `address`).

Viewbox semantics (Nominatim): the `viewbox` query param is a comma-separated tuple `'<minLng>,<maxLat>,<maxLng>,<minLat>'` (longitude-latitude pairs of NW and SE corners). When `opts.bias = { lat, lng }` is provided, generate a ±0.3°-wide box around the city centroid:
- `viewbox=${lng - 0.3},${lat + 0.3},${lng + 0.3},${lat - 0.3}`
- Also set `bounded=1` so Nominatim treats it as a hard filter, not a soft preference.

Nominatim reverse endpoint: `https://nominatim.openstreetmap.org/reverse` with query params `format=json`, `lat=${lat}`, `lon=${lng}`, `addressdetails=1`. Same User-Agent header policy. Returns a single object (not an array) with `display_name` + `address`.

AbortController pattern for both functions (Node ≥22 native fetch supports `signal`):
```
const controller = new AbortController();
const timeoutId = setTimeout(() => controller.abort(), 5000);
try {
  const response = await fetch(url, { signal: controller.signal, headers: {...} });
  ...
} catch (err) {
  if (err.name === 'AbortError') {
    console.warn('Geocoding aborted (5s timeout)');
    return null;
  }
  console.error('Geocoding error:', err.message);
  return null;
} finally {
  clearTimeout(timeoutId);
}
```
