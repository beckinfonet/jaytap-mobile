POST /api/moderation/listings/:id/media — request shape:
- Multipart form-data fields:
  - `photos[]` — up to 40 files; MIME ∈ PHOTO_MIMES (jpeg|png|webp); 25MB/file.
  - `videos[]` — up to 5 files; MIME ∈ VIDEO_MIMES (mp4|quicktime); 25MB/file.
  - `tourUrl` (text) — optional; must be a valid `https://...` URL or empty.
- Total per-request: 45 files / 25MB-per-file / Express body parser limit `'50mb'`.
- Headers: `Authorization: Bearer <firebase-id-token>` (mounted middleware).

POST response codes:
- `200` — atomic update applied; full updated `Property` doc returned.
- `400 MEDIA_INVALID_TYPE` — multer fileFilter rejection.
- `400 MEDIA_INVALID_TOUR_URL` — tourUrl is non-https or malformed.
- `400 MEDIA_NO_OP` — request had zero photos AND zero videos AND no tourUrl.
- `400 MEDIA_TOO_MANY_FILES` — multer LIMIT_FILE_COUNT.
- `400 MEDIA_FILE_TOO_LARGE` — multer LIMIT_FILE_SIZE.
- `401` — no token.
- `403 insufficient-role` — token belongs to a non-mod user.
- `404` — property not found.
- `409 ALREADY_MODERATED` — listing is no longer in `pending` or `rejected` status (e.g., already `live`/`archived`).
- `500` — unexpected error.

DELETE /api/moderation/listings/:id/media?url=<encoded>&kind=<photo|video> — request shape:
- Query string: `url` (required, URL-encoded) + `kind` (`'photo'` or `'video'`).
- Headers: same Bearer token.

DELETE response codes:
- `200` — atomic `$pull` applied (idempotent); full updated `Property` doc returned.
- `400 BAD_REQUEST` — missing/invalid `url` or `kind`.
- `401` / `403` — same as POST.
- `404` — property not found.
- `500` — unexpected error.
- (No 409 for DELETE — `$pull` of a non-member is a no-op; concurrent same-URL deletes are idempotent.)

    ```javascript
    // Phase 3 — POST /api/moderation/listings/:id/media
    // Mod-only (router-level guard at line 57). Multipart upload of photos + videos + tourUrl.
    // Race-safe append via findOneAndUpdate $push; ModerationLog audit with count-only diff.
    router.post(
      '/listings/:id/media',
      mediaUpload.fields([
        { name: 'photos', maxCount: 40 },
        { name: 'videos', maxCount: 5 },
      ]),
      async (req, res, next) => {
        try {
          const photoUrls = (req.files?.photos || []).map(f => f.location);
          const videoUrls = (req.files?.videos || []).map(f => f.location);
          const tourUrlRaw = req.body?.tourUrl;

          // D-08 — generic https:// validation; NO Matterport allowlist.
          // Mitigates T-05 (javascript: scheme injection).
          let tourUrl;
          if (typeof tourUrlRaw === 'string' && tourUrlRaw.length > 0) {
            try {
              const u = new URL(tourUrlRaw);
              if (u.protocol !== 'https:') throw new Error('non-https');
              tourUrl = tourUrlRaw;
            } catch {
              return res.status(400).json({ code: 'MEDIA_INVALID_TOUR_URL', message: 'tourUrl must be a valid https:// URL' });
            }
          }

          // No-op guard
          if (!photoUrls.length && !videoUrls.length && tourUrl === undefined) {
            return res.status(400).json({ code: 'MEDIA_NO_OP', message: 'Provide at least one photo, video, or tourUrl' });
          }

          // Capture before-state for ModerationLog count-only diff (Discretion #7)
          const beforeDoc = await Property.findById(req.params.id, { media: 1 }).lean();
          if (!beforeDoc) return res.status(404).json({ message: 'Property not found' });

          // Build update operators
          const update = {};
          if (photoUrls.length || videoUrls.length) {
            update.$push = {};
            if (photoUrls.length) update.$push['media.photos'] = { $each: photoUrls };
            if (videoUrls.length) update.$push['media.videos'] = { $each: videoUrls };
          }
          if (tourUrl !== undefined) {
            update.$set = { 'media.tourUrl': tourUrl };
          }

          // Race-safe — filter on status mirrors existing M2 atomic-update pattern
          const result = await Property.findOneAndUpdate(
            { _id: req.params.id, status: { $in: ['pending', 'rejected'] } },
            update,
            { new: true, runValidators: true }   // runValidators triggers Plan 03-01 tourUrl validator
          );
          if (!result) {
            return res.status(409).json({ code: 'ALREADY_MODERATED', message: 'Listing is no longer in a state that accepts media curation.' });
          }

          // Audit log — D-15: actorUid sourced EXCLUSIVELY from req.firebaseUid (NEVER req.body / req.headers)
          try {
            await ModerationLog.create({
              actorUid: req.firebaseUid,
              action: 'media-upload',
              targetType: 'property',
              targetId: result._id,
              before: {
                'media.photos.length': beforeDoc.media?.photos?.length || 0,
                'media.videos.length': beforeDoc.media?.videos?.length || 0,
                'media.tourUrl.set':   !!beforeDoc.media?.tourUrl,
              },
              after: {
                'media.photos.length': result.media?.photos?.length || 0,
                'media.videos.length': result.media?.videos?.length || 0,
                'media.tourUrl.set':   !!result.media?.tourUrl,
              },
              at: new Date(),
            });
          } catch (auditErr) {
            // Orphan-tolerant: state flip is committed; audit miss only logs (mirrors existing /approve pattern)
            console.error(JSON.stringify({
              evt: 'moderation_audit_orphan',
              action: 'media-upload',
              targetId: String(result._id),
              actorUid: req.firebaseUid,
              err: auditErr.message,
              ts: new Date().toISOString(),
            }));
          }
          return res.json(result);
        } catch (err) {
          // Multer error code mapping (Pitfall 6)
          if (err.code === 'MEDIA_INVALID_TYPE') return res.status(400).json({ code: 'MEDIA_INVALID_TYPE', message: 'File type not allowed' });
          if (err.code === 'LIMIT_FILE_COUNT')    return res.status(400).json({ code: 'MEDIA_TOO_MANY_FILES', message: 'Too many files' });
          if (err.code === 'LIMIT_FILE_SIZE')     return res.status(400).json({ code: 'MEDIA_FILE_TOO_LARGE', message: 'File exceeds 25MB' });
          // Mongoose validation error (e.g., tourUrl validator on findOneAndUpdate with runValidators)
          if (err.name === 'ValidationError') return res.status(400).json({ code: 'MEDIA_INVALID_TOUR_URL', message: err.message });
          console.error('Error uploading media:', err);
          next(err);
        }
      }
    );
    ```
