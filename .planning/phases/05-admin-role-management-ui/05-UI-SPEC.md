# Phase 5 ‚Äî UI Design Contract

> Visual and interaction contract for the Admin Role Management UI phase. Generated by gsd-ui-researcher; to be verified by gsd-ui-checker.
>
> Brownfield React Native 0.84 (New Architecture) app. **Phase 5 introduces ZERO new design tokens, ZERO new dependencies, and reuses every visual primitive Phases 2 + 3 + 4 established.** Every contract value below maps to existing `useTheme()` tokens (`src/theme/colors.ts`) and inherits the spacing / typography scales locked in `02-UI-SPEC.md` and `03-UI-SPEC.md`. The executor's job is to apply these tokens consistently using two precedents: (a) Phase 3's `<ModerationQueueScreen>` for the screen-level shell (SafeAreaView + ChevronLeft header + FlatList + sibling-mounted modal + AppState cooldown), and (b) Phase 3's `<RejectListingModal>` for the role-picker modal (chip row + 2-tap confirm + inline error surfacing).

- Update Role submit button (uses `colors.primary` ‚Äî neutral primary action)

**60/30/10 split is honored:** screen backgrounds and modal surfaces dominate (~85% of pixels by area); the role pills (one per row) total <5% of pixels on any single Phase 5 screen; accent appears on at most ~1-3 admin pills per visible list.
