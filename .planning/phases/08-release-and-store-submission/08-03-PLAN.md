<objective>
Wave 2 of Phase 8 — physical-device regression smoke walk + bug-fix loop. This is the BLOCKING phase-exit checkpoint per CLAUDE.md M1 testing bar (manual physical-device QA on iOS + Android).

Walks the four mandatory matrices defined in `08-QA-MATRIX.md` (Plan 08-01 Task 2): Keyboard handling (Matrix 1, ~22 cells), Bottom-nav transitions (Matrix 2, ~60 cells), Listing categories (Matrix 3, 18 cells), Role gating (Matrix 4, 8 cells), plus the optional Universal Link smoke (Matrix 5, 1 cell). Defects beyond the smoke walk's scope go to backlog `999.x` UNLESS the user flags them release-blocking — in which case Phase 8 widens scope to address.

Purpose: v1.0.4 is an update submission, not a first-time submission. The regression matrix is the bug-fix surfacing mechanism for the M1 polish ship — it validates that Phase 1–6 deliverables (keyboard, nav, listing form taxonomy, validation, hospitality) survived intact under the bumped build and on the actual devices users will run. Simulators don't certify Fabric render correctness, real touch-target sizing, deep-link behavior, or bilingual viewport overflow.
