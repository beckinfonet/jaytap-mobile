|----------|-------------|
| client → backend `/api/properties` POST/PUT routes | Untrusted multipart/JSON body crosses here; client is JWKS-authenticated but body contents are owner-controlled — schema validators must catch malformed numeric input even when the route layer is bypassed (e.g., direct DB scripts, future endpoints, `findOneAndUpdate` without `runValidators`). |
| migration scripts / admin shells → MongoDB | Direct `.save()` or `db.listings.insertOne()` paths bypass the route-layer 400; schema-level Mongoose `validate` is the only line of defense. |
