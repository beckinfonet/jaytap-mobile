## D-01 Deviation — Documented (RESEARCH.md 2026-04-29 verification)

**CONTEXT.md D-01 prescribes a 7-step sequence:**
> rotate Atlas → rotate AWS → `git rm --cached .env` → add `.env` to `.gitignore` → check in `.env.example` → Railway env-vars-only → verify boot.

**This plan supersedes that sequence.** Specifically, this plan **SKIPS** the `git rm --cached .env` step and skips the "add `.env` to `.gitignore`" step.

**Why (per RESEARCH.md lines 15 + 1161-1163, dated 2026-04-29):**

> "HF-02's REQUIREMENTS.md text says 'Live MONGO_URI password + AWS S3 credentials currently committed' — this is **factually wrong**. Verified 2026-04-29 via `git log --all -- .env` (returned no commits) and `git ls-files` (`.env` not tracked) in the backend repo. `.env` is in `.gitignore` (line 12) and was never committed."

The verification commands documented in RESEARCH.md line 15 returned:
- `git log --all -- .env` → empty (no commits ever touched the file)
- `git ls-files | grep .env` → empty (file is not tracked)
- `.gitignore:12` → already contains `.env` (no edit needed)

`git rm --cached .env` would error (file not in index) or be a no-op; `.gitignore` already excludes the file. BFG and `git filter-repo` are inapplicable (no history to scrub). Adding "skip" steps is procedurally cleaner than running steps that error or no-op.

**Threat-surface reduction is unaffected:** D-01's stated rationale ("every day they sit there is exposure") was framed assuming the file was committed and visible to anyone who cloned the repo. The actual exposure surface is local-machine + the Atlas/AWS consoles where the credentials were created. Rotating Atlas + AWS + revoking old keys (Steps 1, 2, 5 in this plan) closes that local-machine + console surface entirely. The rotate+revoke path is the complete remediation.

**What this plan still does (full HF-02 closure):**
- Atlas password rotation (Task 1 Step 1)
- AWS IAM key rotation + revoke old (Task 1 Steps 2, 5)
- Railway env vars set + verified boot (Task 1 Steps 3, 4)
- `.env.example` checked in (Task 2)
- `.env` exclusion VERIFIED via `git ls-files` audit (Task 2 — defense-in-depth check that the .gitignore:12 exclusion is still effective)

**Audit trail:** This callout exists so that any reader comparing CONTEXT.md D-01 against this plan understands the deviation is intentional, evidence-based, and procedurally complete.
