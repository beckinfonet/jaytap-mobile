<!-- ROLE-09 final-fallback contract -->
Per REQUIREMENTS.md ROLE-09 (verbatim): "If 403 role-revoked retry STILL returns 403, logout + show EN+RU 'Your access changed. Please sign in again.' toast."

The 403 role-revoked branch in this plan implements TWO levels:

1. **First-pass:** server returns 403 `{code: 'role-revoked'}`. apiClient calls `refreshRoleHook()` (which re-fetches `/api/auth/me`, updating user.backendProfile.userType), then retries the original request once. The expected case is that the retry succeeds — the user's role just changed and the cached state was stale.

2. **Post-retry (still-403):** the retry ALSO returns 403 `{code: 'role-revoked'}`. This means the user's access genuinely was revoked (e.g., demoted from admin to user, then attempted an admin-only action). The interceptor MUST: (a) call `logoutHook()` to clear AsyncStorage + reset AuthContext state, (b) call `toastHook('auth.accessChanged')` to surface the bilingual "Your access changed. Please sign in again." Alert.

The `'auth.accessChanged'` key is a PREFIX. The toast hook in AuthContext concatenates `.title` and `.body` to produce the two strings RN's `Alert.alert(title, body)` expects. Plan 12 Task 1 adds the `auth.accessChanged.title` + `auth.accessChanged.body` locale keys.

    ```typescript
    import axios, { AxiosError, AxiosInstance, InternalAxiosRequestConfig } from 'axios';
    import AsyncStorage from '@react-native-async-storage/async-storage';
    import { AuthService } from './AuthService';

    const PRODUCTION_URL = 'https://jaytap-services-production.up.railway.app/api';

    export const apiClient: AxiosInstance = axios.create({
      baseURL: PRODUCTION_URL,
      timeout: 15000,
    });

    // Request interceptor — attach Bearer
    apiClient.interceptors.request.use(async (config: InternalAxiosRequestConfig) => {
      const token = await AsyncStorage.getItem('userToken');
      if (token) {
        config.headers.set('Authorization', `Bearer ${token}`);
      }
      return config;
    });

    type RetryConfig = InternalAxiosRequestConfig & { _retry?: boolean };

    // Single-flight refresh — N parallel 401s share ONE refresh call
    let refreshPromise: Promise<string | null> | null = null;
    async function refreshIdTokenSingleflight(): Promise<string | null> {
      if (!refreshPromise) {
        refreshPromise = (async () => {
          const refreshToken = await AsyncStorage.getItem('refreshToken');
          if (!refreshToken) return null;
          try {
            const newToken = await AuthService.refreshIdToken(refreshToken);
            return newToken;
          } catch (e) {
            return null;
          } finally {
            refreshPromise = null;
          }
        })();
      }
      return refreshPromise;
    }

    // Hooks registered by AuthContext (avoids services-importing-context coupling).
    // Three hooks now: refreshRole, logout, toast. Toast accepts a locale-key prefix
    // (e.g. 'auth.accessChanged') and AuthContext expands it to title + body strings.
    let refreshRoleHook: (() => Promise<void>) | null = null;
    let logoutHook: (() => Promise<void>) | null = null;
    let toastHook: ((localeKeyPrefix: string) => void) | null = null;
    export function registerAuthHooks(hooks: {
      refreshRole: () => Promise<void>;
      logout: () => Promise<void>;
      toast: (localeKeyPrefix: string) => void;
    }) {
      refreshRoleHook = hooks.refreshRole;
      logoutHook = hooks.logout;
      toastHook = hooks.toast;
    }

    // Response interceptor — 401 token-expired refresh + 403 role-revoked refreshRole+retry +
    // post-retry 403 logout+toast (ROLE-09 final fallback).
    apiClient.interceptors.response.use(
      resp => resp,
      async (error: AxiosError) => {
        const original = error.config as RetryConfig | undefined;
        const status = error.response?.status;
        const code = (error.response?.data as any)?.code;

        // Branch 1: 401 token-expired -> silent refresh -> retry once
        if (original && status === 401 && code === 'token-expired' && !original._retry) {
          original._retry = true;
          const newToken = await refreshIdTokenSingleflight();
          if (!newToken) {
            // Refresh-token itself expired -> hard logout (D-11; toast wired in Plan 12 path)
            await logoutHook?.();
            return Promise.reject(error);
          }
          original.headers!.set('Authorization', `Bearer ${newToken}`);
          return apiClient.request(original);
        }

        // Branch 2: 403 role-revoked -> refreshRole -> retry once.
        // If the retry STILL returns 403, the user's access was genuinely revoked
        // (ROLE-09 final fallback): logout + bilingual "Your access changed" toast.
        if (original && status === 403 && code === 'role-revoked' && !original._retry) {
          original._retry = true;
          await refreshRoleHook?.();
          try {
            return await apiClient.request(original);
          } catch (retryErr: any) {
            // ROLE-09: still-403 after refreshRole = real revocation, not stale cache
            if (retryErr?.response?.status === 403) {
              await logoutHook?.();
              toastHook?.('auth.accessChanged');
            }
            throw retryErr;
          }
        }

        return Promise.reject(error);
      }
    );
    ```
